# Tag 审核规范

**Version**: V0.2
**Date**: 2026-07-03

---

## 1. 审核流程

```
新增 Tag
↓
格式检查（TAG_NAMING.md）
↓
分类检查（CATEGORY.md）
↓
去重检查（canonical_tags.csv）
↓
Alias 检查（aliases.csv）
↓
审核通过 → 更新数据
```

---

## 2. 审核检查项

### 2.1 格式检查
- [ ] 简体中文（英文标签除外）
- [ ] 长度 2-8 字
- [ ] 无书名号、引号等符号
- [ ] 英文标签已大写

### 2.2 分类检查
- [ ] 属于已有的一级维度
- [ ] 不与同维度标签重叠
- [ ] 已标注互斥标签（如有）

### 2.3 去重检查
- [ ] 不存在完全相同的标准标签
- [ ] 相似度 > 0.8 的标签已合并
- [ ] 已检查别名表，避免重复

### 2.4 Alias 检查
- [ ] 已添加所有已知别名
- [ ] 别名类型正确（synonym/close/platform/abbr/slang）
- [ ] 一个别名只指向一个标准标签

---

## 3. 审核频率

| 类型 | 频率 | 说明 |
|------|------|------|
| 用户新增 | 实时 | 用户添加标签后自动进入待审核 |
| 平台导入 | 每月 | 批量导入前人工审核 |
| 社区提取 | 每周 | 从推书帖提取的高频词审核 |
| 定期清理 | 每季度 | 清理低频标签和冲突标签 |

---

## 4. 审核结果

| 结果 | 操作 |
|------|------|
| **通过** | 添加到 canonical_tags.csv，关联 aliases.csv |
| **修改** | 返回修改建议，重新提交 |
| **拒绝** | 说明理由，记录到 CHANGELOG.md |
| **合并** | 与已有标签合并，添加为别名 |

---

## 5. 常见拒绝原因

| 原因 | 说明 |
|------|------|
| 格式不符 | 不符合 TAG_NAMING.md 规范 |
| 与现有标签重叠 | 已有同义或高度相似的标签 |
| 含义模糊 | 无法明确定义标签范围 |
| 使用频率过低 | 预期使用次数 < 5 |
| 与平台标签冲突 | 与主流平台的标签体系不一致 |
