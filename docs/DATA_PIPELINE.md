# 数据处理管线：书荒雷达

**Version**: V0.1
**Date**: 2026-07-03

---

## 1. 概述

数据从两个入口进入系统：
1. **手动录入** — 用户在表单中输入
2. **文本导入** — 粘贴推书文本，自动解析

---

## 2. 手动录入流程

```
用户填写表单 → POST /api/books → upsertBook() → addTags() → 返回结果
```

### 字段映射

| 表单字段 | 数据库字段 | 说明 |
|----------|-----------|------|
| title | title | 必填，自动去除《》 |
| author | author | 默认"待补充" |
| platform | platform | 默认"待补充" |
| summary | summary | 简介/推荐语 |
| officialUrl | official_url | 正版链接 |
| status | status | 想看/在看/已看/避雷 |
| officialTags | → book_tags (type=official) | 官方标签 |
| userTags | → book_tags (type=user) | 用户标签 |

---

## 3. 文本导入流程

```
粘贴文本 → POST /api/import/text → 哈希去重 → 解析书目 → 入库 → 建立推荐边
```

### 3.1 去重机制

1. 计算文本 SHA-256 哈希
2. 查询 `import_batches` 是否已存在
3. 已存在 → 返回"这段文本已经导入过"
4. 不存在 → 继续解析

### 3.2 文本解析规则

**书名识别**：正则 `/《([^》]{1,80})》/g`

**字段提取**（每个书名块内）：

| 标签 | 正则 | 说明 |
|------|------|------|
| 作者 | `作者\s*[:：]\s*([^\n]+)` | |
| 平台 | `平台\s*[:：]\s*([^\n]+)` | |
| 官方标签 | `官方标签\s*[:：]\s*([^\n]+)` | |
| 标签 | `标签\s*[:：]\s*([^\n]+)` | 也支持 `#hashtag` 格式 |
| 简介 | `简介\s*[:：]\s*([^\n]+)` | 也匹配"推荐语" |
| 链接 | `https?://\S+` | |

**平台猜测**：文本中包含"晋江/起点/番茄/长佩/豆瓣阅读/知乎盐选/刺猬猫"时自动识别。

### 3.3 推荐边创建

同一批次导入的所有书两两建立推荐关系：

```
bookIds = [1, 2, 3]
→ (1,2), (1,3), (2,3)  // C(n,2) 条边
→ weight = 1
→ 重复导入时 weight + 1
```

---

## 4. 导入文本格式

### 4.1 标准格式

```
《书名》
作者：作者名
平台：晋江
官方标签：幻想未来，情有独钟
标签：bg，老实人女主
简介：这是一本好书...
```

### 4.2 Hashtag 格式

```
《书名》
#bg #老实人 #校园
这是一本好书...
```

### 4.3 混合格式

```
《书A》
作者：作者1
标签：#bg #校园

《书B》
平台：起点
官方标签：都市，异能
```

## 5. 数据质量检查（计划 V0.2）

| 检查项 | 规则 | 处理 |
|--------|------|------|
| 空书名 | title 为空 | 拒绝导入 |
| 重复书名 | UNIQUE(title, author) | UPSERT 合并 |
| 标签不足 | 标签数 < 3 | 标记警告 |
| 噪声标签 | 频次 = 1 | 标记待审核 |

## 6. 爬虫数据管线（V0.1 已实现）

```
晋江爬虫 → JSON 文件 → 导入脚本 → SQLite
```

### 当前实现（2026-07-03）

- **爬虫**: Scrapy 框架，基于 [jjwxc-crawler](https://github.com/dev-chenxing/jjwxc-crawler) 改造
- **数据量**: 1125 本独立小说元数据（已去重合并）
- **数据位置**: `data/novels_jjwxc.jsonl`（单个JSONL文件，2.51MB）
- **进度文件**: `crawler/data/raw/jjwxc/crawl_progress_unified.json`
- **限制**: bookbase.php 需要登录，当前只能爬取游客可访问的部分

### 爬虫文件

| 文件 | 用途 |
|------|------|
| `crawler/original/jjcrawler/jjcrawler/spiders/novel_metadata.py` | 全分类爬虫（4250种组合） |
| `crawler/original/jjcrawler/jjcrawler/spiders/novel_metadata_yanqing.py` | 言情专用爬虫 |

### 数据来源优先级

| 优先级 | 来源 | 内容 | 状态 |
|--------|------|------|------|
| 一级 | 平台公开元数据 | 标题、作者、标签、分类、简介、字数、状态、链接 | ✅ 已爬取1125本 |
| 二级 | 公开推书内容 | 小红书、豆瓣、博客、论坛 | ⏳ 计划中 |
| 三级 | 用户个人收藏 | Obsidian、TXT、CSV、JSON | ✅ 支持文本导入 |
| 四级 | AI 离线补充 | 仅规则无法识别时调用 | ⏳ 计划中 |

### AI 使用原则：Rule First, AI Last

1. 官方标签
2. 分类信息
3. 规则匹配
4. 推荐帖关键词
5. **最后**：AI 补标签（仅离线）

用户每次搜索 **0 Token 消耗**。
